# CUDA Batch Image Processor
### CUDA at Scale for the Enterprise — Course Project

---

## 1. What I Did

I built a GPU-accelerated batch image processing pipeline that runs three stages on every input image, entirely on the GPU:

| Stage | Method | GPU API |
|---|---|---|
| Gaussian blur | Smooths noise before edge detection | NPP `nppiFilterGauss_8u_C1R` |
| Histogram equalization | Stretches contrast so dark images pop | **My own CUDA kernels** (`buildHistogram_kernel`, `applyLUT_kernel`) |
| Sobel edge detection | Highlights object boundaries | NPP `nppiFilterSobelHorizBorder_8u16s_C1R` + `nppiMagnitude_16s8u_C1R` |

The program processes **120 synthetic grayscale PGM images** (256×256 px each) in one batch — satisfying the "hundreds of small inputs" requirement.

### Why these three stages?
- Gaussian blur first removes sensor noise that would create false edges.
- Histogram equalization compensates for images that are too dark or too bright (makes Sobel more robust).
- Sobel is the classic first step in computer vision edge detection pipelines.

### My Custom Kernels (the required CPU-free GPU computation)
I wrote two CUDA kernels in `src/batchImageProcess.cu`:

1. **`buildHistogram_kernel`** — Each CUDA thread reads one pixel and uses `atomicAdd` to increment the matching intensity bin in a 256-element array on the GPU.
2. **`applyLUT_kernel`** — Each thread remaps its pixel through the equalized look-up table computed on the CPU from the histogram CDF.

## 3. Build & Run

### Prerequisites
- CUDA Toolkit ≥ 11.x with NPP libraries (`libnpp*`)
- CUDA Samples installed (for `ImageIO.h`, `helper_cuda.h`, etc.)
- Python 3 + `numpy` (+ `Pillow` optional, for richer test images)

### Steps

```bash
# Clone the repo
git clone https://github.com/<your-username>/cuda-batch-project.git
cd cuda-batch-project

# Generate 120 synthetic test images
python3 scripts/generate_data.py --count 120 --outdir data

# Build
make all

# Run the batch processor
./bin/run --input data --output output

# Or do all three steps in one go:
bash run.sh
```

If the CUDA Samples Common headers are not at `/usr/local/cuda/samples`, override:

```bash
make all CUDA_SAMPLES_PATH=/path/to/cuda-samples
```

## 5. Dataset

I generate the dataset programmatically with `scripts/generate_data.py` rather than downloading from an external source. Each synthetic image contains random rectangles, ellipses, and Gaussian blobs on a noisy background — giving the Sobel detector real edges to find while keeping the project self-contained.

To use your own images instead, copy `.pgm` files into `data/` and re-run:
```bash
./bin/run --input data --output output
```
